### Пример ошибки логики у Gemma 3 4B
Задача: «Склад 500 шт, продажи 20/день. Когда останется < 100?»  
- Правильный ответ: **21 день** (500 − 20×21 = 80 < 100)  
- Gemma 3 4B ответила: **24** ❌  
- Qwen 3 VL 8B: **21** ✅  
- Gemma 4 E4B: ✅ (но нужен большой max_tokens)

## Особенности Gemma 4 E4B

Это **thinking-модель** (аналог DeepSeek R1, o1):
- Перед ответом генерирует цепочку размышлений в поле `reasoning_content`
- При `max_tokens=300` — все 297 уходят на thinking, контент пустой
- **Обязательно:** `max_tokens ≥ 1000`, лучше 1500
- Thinking-токены не отображаются пользователю, но тратят время и бюджет

```python
# Правильный запрос к Gemma 4
{
    "model": "google/gemma-4-e4b",
    "messages": [...],
    "max_tokens": 1500,   # обязательно большой
    "temperature": 0.1    # thinking-модели не нуждаются в высокой temp
}
```

## Оптимальные параметры

### Gemma 3 4B — быстрые операционные задачи
```python
temperature = 0.1   # код, JSON, факты
temperature = 0.7   # текст, идеи
max_tokens  = 200–300
```

### Gemma 4 E4B — сложный reasoning
```python
temperature = 0.1–0.3
max_tokens  = 1000–1500   # КРИТИЧНО
# Время ответа: 30–60 сек — использовать только когда важна точность
```

### Qwen 3 VL 8B — качественный текст и код
```python
temperature = 0.3–0.7
max_tokens  = 200–400
# Поддерживает изображения (передавать base64 в content)
```
